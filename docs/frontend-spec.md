# 前端重构 Spec

## 问题

当前前端使用 Tailwind CDN，在 Docker 环境中 CDN 加载失败导致样式全丢（只剩裸 HTML）。且 UI 设计过于简陋，缺少关键功能（日志窗口、结果展示、下载）。

## 目标

重写前端为**纯内联 CSS**（不依赖任何 CDN），视觉风格参考现代 Dashboard 设计，同时补全功能缺失。

---

## 页面结构

### 1. 登录页

- 居中卡片，毛玻璃效果背景
- Logo + 标题「regf-works」
- 用户名 / 密码输入框（圆角、带图标）
- 登录按钮（渐变色，hover 动效）
- 错误提示（红色 toast）

### 2. 主界面布局

```
┌─────────────────────────────────────────────────┐
│ Header: Logo + 用户名 + 退出按钮               │
├──────────┬──────────────────────────────────────┤
│ 侧边栏   │ 主内容区                             │
│          │                                      │
│ [注册]   │ ┌─────────────────────────────────┐  │
│ [结果]   │ │  当前选择的面板内容              │  │
│ [设置]   │ │                                 │  │
│          │ └─────────────────────────────────┘  │
│          │                                      │
│          │ ┌─────────────────────────────────┐  │
│          │ │  日志窗口（固定底部，可折叠）     │  │
│          │ └─────────────────────────────────┘  │
└──────────┴──────────────────────────────────────┘
```

### 3. 注册面板

- 平台选择：Grok / Fireworks 两个 tab 切换
- 表单区：代理输入、数量选择、邮箱 provider 选择
- 「开始注册」按钮（带 loading 动画）
- 「停止」按钮（注册进行中可点击）

### 4. 日志窗口

- 固定在内容区底部，高度 200px，可拖拽调整
- 等宽字体，暗色背景
- 自动滚动到底部
- 右上角：清空按钮 + 折叠按钮
- 不同日志类型颜色区分：
  - `[+]` 成功 → 绿色
  - `[-]` 失败 → 红色
  - `[*]` 进行中 → 蓝色
  - `[!]` 警告 → 黄色

### 5. 结果面板

- 表格展示所有注册结果：
  - 列：平台、邮箱、状态、Token/APIKey（截断显示）、时间
- 每行操作：
  - 「复制」按钮 → 复制完整凭据 JSON
  - 「查看」按钮 → 展开详情弹窗
- 顶部功能栏：
  - 「导出全部」按钮 → 下载 JSON 文件
  - 「清空」按钮
  - 筛选：平台 / 状态

### 6. 设置面板

三个折叠卡片：

**邮箱配置**
- Provider 优先级（下拉多选或文本）
- AHEM: base_url + domains
- YYDS: base_url + api_key（密码输入框）

**代理配置**
- 默认代理
- 代理池（textarea，每行一个）

**账号安全**
- 当前用户名（只读显示）
- 新用户名
- 新密码 + 确认密码
- 修改按钮（红色警告色）
- 修改后自动退出

## 技术要求

1. **纯内联 CSS** — 不依赖任何 CDN（Tailwind、Bootstrap 都不用）
2. **单 HTML 文件** — 所有 CSS + JS 内联，Go embed 打包
3. **响应式** — 移动端侧边栏折叠为底部 tab
4. **无框架** — 纯 Vanilla JS
5. **结果持久化** — 存 localStorage，刷新不丢失
6. **导出功能** — JSON 文件下载（Blob URL）

---

## 交互细节

1. 注册进行中，按钮变为 loading 状态（旋转图标），不可重复点击
2. 结果实时出现在结果面板（不需要手动切换）
3. 日志窗口注册完成后显示总结行（成功 x 个，失败 x 个）
4. 复制按钮点击后变为「已复制」并 1.5s 后恢复
5. 设置保存后显示 toast 提示（3s 自动消失）
6. 退出登录清空 localStorage + 结果缓存
